Image usage instructions

Docker image pull တွေကို မြန်ဆန်စေဖို့အတွက် အောက်ပါ command ကို အသုံးပြုနိုင်ပါတယ် registry mirror:

sudo tee /etc/docker/daemon.json <<EOF
{
    "registry-mirrors": ["https://docker.gwinhub.qzz.io"]
}
EOF
# Configure လုပ်ပြီးသွားရင် Docker service ကို restart လုပ်ရပါမယ်။
sudo systemctl restart docker

မတူညီသော image repositories များမှ image များကို ဆွဲယူရန် ဤ proxy ကို အသုံးပြုရန်အတွက် အောက်ပါ command များကို ကြည့်ပါ။：


# docker pull nginx:latest
docker pull docker.gwinhub.qzz.io/library/nginx:latest  # Pull the official Docker image

# docker pull quay.io/coreos/etcd:latest
docker pull quay.gwinhub.qzz.io/coreos/etcd:latest  # Pull Quay Mirror

# docker pull gcr.io/google-containers/busybox:latest
docker pull gcr.gwinhub.qzz.io/google-containers/busybox:latest  # 拉取 GCR 镜像

# docker pull k8s.gcr.io/pause:latest
docker pull k8s-gcr.gwinhub.qzz.io/pause:latest  # 拉取 k8s.gcr.io 镜像

# docker pull registry.k8s.io/pause:latest
docker pull k8s.gwinhub.qzz.io/pause:latest  # 拉取 registry.k8s.io 镜像

# docker pull ghcr.io/github/super-linter:latest
docker pull ghcr.gwinhub.qzz.io/github/super-linter:latest  # 拉取 GitHub 容器镜像

# docker pull docker.cloudsmith.io/public/repo/image:latest
docker pull cloudsmith.gwinhub.qzz.io/public/repo/image:latest  # 拉取 Cloudsmith 镜像

အလုပ်သမားအသုံးပြုမှု ကုန်ဆုံးခြင်းကို ရှောင်ရှားရန်အတွက် ပုံကို ကိုယ်တိုင်ဆွဲယူနိုင်ပြီး ပြန်လည် tag လုပ်ကာ သင့်ဒေသခံ image repository သို့ push လုပ်နိုင်ပါသည်။
